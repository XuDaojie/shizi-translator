# Shizi Windows 桌面图标设计规格

## 目标

为 Shizi 设计一枚适用于 Windows 任务栏、开始菜单、桌面快捷方式和安装包的应用图标。图标需要在 16-32 px 小尺寸下仍能辨认，并延续现有「文 / A」翻译字标的品牌记忆。

## 最终方案

采用用户确认的「成熟主导」方案：极简圆角方形、标准「文」字、拉丁字母 A、32 px 及以上尺寸使用的短双向箭头，以及参考柿子成熟过程的自然渐变背景。

### 图形结构

- 画布为圆角方形，圆角约为边长的 22%。
- 左侧使用标准简体中文「文」字，不做抽象变形；字形参考 Windows 系统字体 `Microsoft YaHei Bold`。
- 右下使用白色大写 A，字形参考 `Segoe UI Bold`。
- 32 px 及以上尺寸在「文」与 A 之间放置两条短双向箭头，表达语言之间可以相互转换。
- 「文」和 A 使用纯白色；箭头使用浅奶黄色 `#FFF3DF`，避免与主字标粘连。
- 保持足够留白，不增加描边、阴影、气泡、柿子轮廓、叶片或其他水果图形。

### 背景配色

背景只参考柿子在不同成熟阶段的颜色，不表现水果本身。采用左上至右下的整面自然渐变：

| 阶段 | 色值 | 渐变位置 |
| --- | --- | --- |
| 青柿黄绿 | `#73852E` | 0% |
| 转色金橙 | `#D99624` | 24% |
| 熟柿橙红 | `#E75B25` | 62% |
| 深熟红橙 | `#BD371D` | 100% |

渐变不压缩成字间色带，整体保持自然、连续的成熟过程。视觉重心以橙红色为主，青绿色仅用于建立颜色变化和品牌辨识度。

## 小尺寸规则

- 16、20、24 px：共用小尺寸专用构图，保留完整四段成熟渐变，删除无法稳定辨认的双向箭头；相对大尺寸构图，「文」放大约 15%，A 放大约 14%，并保留外缘和字符间至少 1 个实际像素。三个尺寸分别从小尺寸 SVG 独立渲染，不互相缩放。
- 32-48 px：完整显示标准字形和短双向箭头。
- 64 px 以上：保持相同构图，不追加高光、纹理或立体效果。
- 所有尺寸应独立检查，不能仅依赖 1024 px 图自动缩放后的效果。

## 交付物

后续实现阶段生成：

- 两份可编辑 SVG 源文件：32 px 及以上使用的完整构图，以及 16/20/24 px 使用的无箭头小尺寸构图。
- 从完整构图 SVG 导出的 `1024x1024` 透明 PNG 主图。
- 使用项目现有 Tauri CLI 2.11.4 的 `tauri icon` 命令，从主 SVG 生成标准平台图标和 32、48、64、128、256 px 等标准 Windows 帧。
- 从小尺寸 SVG 分别生成 16、20、24 px PNG；最终 Windows `icon.ico` 精确组装 16、20、24、32、48、64、128 和 256 px 帧，其中 16/20/24 px 不得由主图缩放。
- 替换 `src-tauri/icons/icon.ico`，不改变现有 Tauri 图标配置路径。

## 生成流程

1. 保留完整构图 SVG 和小尺寸 SVG 作为事实来源。
2. 使用 `tauri icon --png 1024` 从完整构图 SVG 导出 1024 px PNG；该 PNG 用于交付与检查，不再作为其他尺寸的中间输入，避免二次缩放。
3. 使用 `tauri icon --png` 直接从完整构图 SVG 渲染 32、48、64、128、256 px 等标准帧。
4. 使用相同命令直接从小尺寸 SVG 渲染 16、20、24 px 帧。
5. 将两组 PNG 组装为最终 ICO。Tauri CLI 2.11.4 默认 ICO 仅包含 16、24、32、48、64、256 px，且所有帧来自同一源图；因此不能直接使用默认 ICO，需要覆盖其中的 16/24 px 并补入 20 px。
6. 组装后读取 ICO 目录并逐项验证尺寸集合及每帧内容，再替换项目现有 `icon.ico`。

## 验收标准

- 「文」必须是标准、可读的简体中文字形。
- 16、20、24 px 下仍能区分「文」和 A；32 px 起还能区分双向转换关系。
- 白色字标在渐变各区域上保持清晰，不出现明显糊边或颜色渗漏。
- 圆角、字标和箭头在各尺寸中不被裁切。
- 在 Windows 浅色与深色桌面背景、任务栏和开始菜单中均可辨认。
- 图标中不得出现柿子或其他水果造型；柿子仅作为颜色参考。

## 范围外

- 不调整应用界面主题色。
- 不设计完整品牌手册、文字 Logo 或宣传物料。
- 不在本阶段替换应用图标文件；实际生成、导出和集成进入后续实现计划。
